# Shell Scripting Lab

## Objective

Learn basic shell scripting concepts used in Linux administration and DevOps automation.

## Scripts Created

### hello.sh

Purpose:
Print messages using echo command.

Concepts Learned:

* Shebang (#!/bin/bash)
* echo command
* Script execution

### variables.sh

Purpose:
Store and display values using variables.

Concepts Learned:

* Variables
* String substitution
* Reusable scripts

### input.sh

Purpose:
Accept user input from terminal.

Concepts Learned:

* read command
* User interaction
* Dynamic scripting

## Commands Used

chmod +x filename.sh
./filename.sh
cat filename.sh
vim filename.sh

## Key Learnings

* Linux scripts automate repetitive tasks.
* Execute permission is required before running scripts.
* Variables make scripts reusable.
* User input helps create dynamic automation.

## Real DevOps Usage

Shell scripts are commonly used for:

* Server provisioning
* Backup automation
* Log cleanup
* Deployment tasks
* Monitoring scripts
